# Chapter 5 — People Oriented Methodologies

---

## 1 Introduction to People Oriented Methodologies

People oriented methodologies focus on the **human aspect of system development**.

Traditional development approaches focus mainly on system structure or development processes. People oriented methodologies emphasize how **users interact with the system and how human behavior affects system design**.

These methodologies recognize that successful systems must consider the needs, abilities, and behavior of the people who use them.

Human factors such as user skills, emotions, work environment, and social behavior are important when designing software systems.

---

## 2 Socio Technical Systems

A socio technical system is a system that combines **social elements and technical elements**.

The social system includes the people, organizational culture, and work practices.  
The technical system includes hardware, software, and technical infrastructure.

For a system to function effectively, both parts must work together.

| Component | Description |
|---|---|
| Social system | People, roles, communication, and organizational behavior |
| Technical system | Technology, software, hardware, and infrastructure |

Example

Hospital Information System

Social part  
Doctors, nurses, administrators

Technical part  
Medical databases, software interfaces, hospital network

If the technical system is complex and difficult to use, medical staff may avoid using it, which reduces system effectiveness.

---

## 3 Importance of Understanding Users

People oriented methodologies emphasize studying the characteristics of system users.

Understanding users helps developers design systems that are easier to use and more effective.

User characteristics include:

• Age  
• Education level  
• Computer literacy  
• Physical abilities  
• Language preferences  
• Work environment

Different users have different needs and expectations.

Example

A system designed for experienced IT professionals may not be suitable for elderly users with limited technical knowledge.

## 4 Types of Systems That Require User Study

Some systems require extensive study of user behavior before development.

| System Type | Explanation |
|---|---|
| Safety critical systems | Systems where human error may cause serious damage |
| Collaborative systems | Systems used by groups of people working together |
| Public facing systems | Systems used by large and diverse populations |
| Decision support systems | Systems used by experts to make important decisions |

## 5 Examples of People Oriented Systems

### Hospital Information System

Users include doctors, nurses, and administrative staff.

Doctors require fast access to patient data during emergencies.  
Nurses may need workflow tracking for patient care.  
Administrators require accurate billing information.

If the interface is complicated, medical staff may avoid using the system, which can cause serious problems.

---

### Air Traffic Control System

Air traffic controllers work under extreme pressure.

The system must present information clearly and minimize distractions.

Important user characteristics include:

• High concentration levels  
• Fast decision making  
• Ability to handle multiple tasks simultaneously

---

### E Commerce Platforms

Users include people of different ages, cultures, and technical skills.

Developers must consider:

• Trust in online payment  
• User interface simplicity  
• Page loading speed

Understanding user behavior helps increase customer satisfaction.

---

### Learning Management Systems

Educational platforms must consider different learning styles.

Students may prefer:

• Visual learning  
• Text based learning  
• Interactive content

Accessibility is also important for users with disabilities.

## 6 Persona in System Design

A persona is a **fictional representation of a typical system user** created using real user data.

Personas help developers understand user needs and design systems that match user behavior.

---

## 7 Purpose of Creating a Persona

Personas are created to represent the characteristics of typical users.

They help developers make design decisions based on user needs.

Benefits of personas include:

• Better understanding of user behavior  
• Improved system usability  
• Better interface design  
• Focus on target audience

---

## 10 Soft Systems Methodology (SSM)

Soft Systems Methodology is a problem solving approach used for complex systems involving human activities.

Unlike traditional development methods, SSM focuses on understanding the **problem situation from multiple perspectives**.

SSM is useful when problems are unclear or difficult to define.

---

## 11 Stages of Soft Systems Methodology

SSM usually involves several stages.

| Stage | Description |
|---|---|
| Problem situation analysis | Study the real world situation |
| Rich picture creation | Visualize the problem environment |
| Root definition | Define the purpose of the system |
| Conceptual model development | Create models of possible solutions |
| Comparison with real world | Evaluate proposed solutions |
| Implementation | Apply improvements to the system |

SSM helps organizations understand complex problems before developing solutions.

---

## 12 Modeling Real World Systems

People oriented methodologies involve converting real world activities into software systems.

This process is called modeling.

Steps involved in modeling include:

1 Observe real world behavior  
2 Identify important activities  
3 Convert activities into system functions  
4 Link functions into workflows  
5 Build the software system

After development, the system is evaluated to determine whether it solves the original problem.

## 15 IS Methodology vs Software Development Framework

Many people confuse methodologies and frameworks, but they are different concepts.

| Feature | IS Methodology | Software Development Framework |
|---|---|---|
| Definition | A structured process that describes how a system should be developed | A structure used to organize thinking and problem solving |
| Purpose | Provides step by step instructions for development | Provides guidelines for understanding problems |
| Structure | Has specific stages and procedures | Provides a conceptual structure |
| Example | XP, RAD, Scrum | CATWOE framework |

Important concept

A methodology tells developers **how to develop the system**, while a framework helps developers **analyze and understand the problem**.

## 17 Process of Modeling Real World into Software

System developers must convert real world activities into system functions.

This process is called **modeling**.

Steps involved in modeling

| Step | Description |
|---|---|
| Observe real world behavior | Study how users perform tasks |
| Identify important activities | Select the most relevant actions |
| Convert activities into system functions | Translate tasks into software features |
| Link tasks into processes | Connect functions into workflows |
| Build the information system | Develop software based on these processes |
| Continuous improvement | Evaluate system and improve it |

Example

Real world task

Student submits assignment to lecturer.

Software system representation

1 Student uploads file  
2 System verifies submission  
3 System stores file in database  
4 Lecturer accesses assignment

## Chapter Summary

People oriented methodologies focus on the human aspects of system development and emphasize the importance of understanding how users interact with technology. Unlike process oriented or structured methodologies that mainly focus on development procedures or system structure, people oriented approaches study user behavior, skills, work environment, and social interactions to design more effective systems. Important concepts in this chapter include socio technical systems, user characteristics, persona creation, and Soft Systems Methodology. Developers must observe real world activities and convert these behaviors into software functions through modeling. By understanding the needs and characteristics of users, developers are able to design systems that are easier to use, improve user satisfaction, and ensure that the system functions effectively in real world environments.
