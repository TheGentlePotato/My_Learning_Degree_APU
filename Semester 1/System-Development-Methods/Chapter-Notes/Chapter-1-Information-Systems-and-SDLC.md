# Chapter 1 — Information Systems and System Development Life Cycle

---

## 1. Introduction to Information Systems

An **Information System (IS)** is a combination of technology, data, people, and processes that work together to collect, process, store, and distribute information.

The goal of an information system is to help organizations:

• Manage operations  
• Store and process data  
• Support decision making  
• Improve efficiency and productivity

### Data vs Information

| Term | Meaning | Example |
|-----|-----|-----|
| Data | Raw facts without meaning | Product ID, price, customer name |
| Information | Processed data with meaning | Sales report, purchase history |

Example

Online shopping system

| Stage | Example |
|-----|-----|
| Data | Customer name, product ID, price |
| Processing | System calculates total cost |
| Information | Sales report generated |

---

## 2. Definition of a System

A **system** is a group of related components that work together to achieve a common objective.

Every system consists of three basic elements.

| Element | Description | Example |
|-----|-----|-----|
| Input | Data or resources entering the system | Student marks |
| Processing | Activities that transform input | Calculate average score |
| Output | Result produced by the system | Final grade report |

Example

Student grading system

Input → Student marks  
Processing → Calculate grade  
Output → Grade report

---

## 3. Components of an Information System

An information system contains **five main components**.

| Component | Description | Examples |
|-----|-----|-----|
| Hardware | Physical devices used by the system | Servers, laptops, routers |
| Software | Programs that run on hardware | Operating system, applications |
| Data | Raw facts stored in system | Student records, sales data |
| People | Users and IT professionals | Students, developers |
| Processes | Procedures used to operate system | Login process, payment process |

---

## 10. Information Systems Development

Organizations constantly develop new systems or improve existing systems.

Reasons include

• New business requirements  
• Technology upgrades  
• System inefficiency  
• Increased user demand

To manage development properly, organizations use the **System Development Life Cycle (SDLC)**.

---

## 11. System Development Life Cycle (SDLC)

SDLC is a structured process used to develop information systems.

### Main SDLC Phases

| Phase | Purpose |
|-----|-----|
| Planning | Identify problem and feasibility |
| Analysis | Gather requirements |
| Design | Create system blueprint |
| Implementation | Build and test system |
| Maintenance | Update and support system |

---

## 12. Planning Phase

The planning phase identifies the **problem and project feasibility**.

### Activities

• Identify system problems  
• Define project scope  
• Evaluate solutions  
• Estimate resources and cost

### Inputs and Outputs

| Input | Output |
|-----|-----|
| Problem statement | Project proposal |
| Current system issues | Feasibility report |
| Business needs | Initial study report |

Example

Company identifies inefficient inventory tracking and proposes a new inventory system.

---

## 13. Analysis Phase

The analysis phase focuses on **understanding system requirements**.

### Activities

• Study current system  
• Identify user needs  
• Gather requirements  
• Document system specifications

### Fact Finding Techniques

| Method | Description |
|-----|-----|
| Interview | Direct discussion with users |
| Questionnaire | Written questions for many users |
| Observation | Watch users perform tasks |
| Document review | Study existing reports and manuals |

### Output

System Requirements Specification (SRS)

---

## 14. Design Phase

The design phase converts requirements into a **technical system blueprint**.

### Design Activities

• Design system architecture  
• Design database structure  
• Design user interface  
• Design program modules  
• Design security controls

| Input | Output |
|-----|-----|
| SRS document | Design specification |

---

## 15. Implementation Phase

The implementation phase involves **building the system**.

### Activities

• Write program code  
• Create databases  
• Integrate system modules  
• Test the system

### Output

Fully operational system

---

## 16. Maintenance Phase

After deployment, the system enters the maintenance phase.

### Maintenance Activities

• Fix bugs  
• Improve system performance  
• Add new features  
• Update system technology

Maintenance ensures long term system operation.

---

## 17. Continuous Improvement in SDLC

SDLC forms a **continuous cycle**.

After maintenance, organizations often identify new improvements.

This leads back to the **planning phase** for system updates.

## 19. Importance of Using SDLC

Using SDLC provides several advantages.

• Clear project structure  
• Higher software quality  
• Better project control  
• Reduced development risk

Without SDLC

• Poor planning  
• Unclear requirements  
• Development delays  
• System failure

## Chapter Summary

An information system consists of **hardware, software, data, people, and processes** working together to produce useful information.

Organizations rely on information systems to manage operations and support decision making.

The **System Development Life Cycle** provides a structured method to develop systems through planning, analysis, design, implementation, and maintenance.
