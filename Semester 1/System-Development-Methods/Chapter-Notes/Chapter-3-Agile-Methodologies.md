# Chapter 3 — Agile Methodologies

---

## 1 Introduction to Agile Methodologies

Agile methodologies are modern approaches used to develop software in a flexible and adaptive way. Unlike traditional development methods that require detailed planning before development begins, Agile allows requirements and solutions to evolve during the project.

Agile focuses on collaboration, customer feedback, and rapid delivery of working software.

Traditional methodologies such as Waterfall follow a sequential process where each phase must be completed before the next phase begins. Agile takes a different approach by dividing development into small cycles called iterations.

Each iteration produces a working part of the system that users can test and evaluate.

---

## 2 Why Agile Was Introduced

Traditional software development methods faced several problems.

| Problem | Explanation |
|---|---|
| Changing requirements | Customers often change their requirements during development |
| Long development cycles | Users may wait months or years before seeing working software |
| Late error detection | Problems discovered late are expensive to fix |
| Limited customer involvement | Customers only participate in early project stages |

Agile methodologies were introduced to solve these problems by allowing continuous improvement and customer involvement throughout the development process.

---

## 3 Key Characteristics of Agile Development

Agile development has several important characteristics.

| Characteristic | Explanation |
|---|---|
| Iterative development | Software is developed in repeated cycles |
| Incremental delivery | Small system features are delivered step by step |
| Continuous feedback | Users review the system regularly |
| Adaptive planning | Plans change based on feedback |
| Team collaboration | Developers and stakeholders communicate frequently |

These characteristics allow development teams to respond quickly to changes.

---

## 4 Agile Manifesto

The Agile Manifesto was created in 2001 by a group of software developers who wanted to improve software development practices.

The manifesto defines four core values.

| Agile Value | Meaning |
|---|---|
| Individuals and interactions over processes and tools | Communication between people is more important than strict procedures |
| Working software over comprehensive documentation | Delivering working software is more valuable than producing large documentation |
| Customer collaboration over contract negotiation | Developers work closely with customers to build the right product |
| Responding to change over following a plan | Agile teams adapt to changing requirements |

These values emphasize flexibility, collaboration, and delivering working software.

---

## 5 Agile Manifesto Principles

The Agile Manifesto is supported by **12 principles** that guide Agile software development.

| No | Agile Principle | Explanation |
|---|---|---|
| 1 | Customer satisfaction through early and continuous delivery of valuable software | Deliver useful software early and improve it continuously |
| 2 | Welcome changing requirements even late in development | Changes improve the system and should be accepted |
| 3 | Deliver working software frequently | Short development cycles ensure faster delivery |
| 4 | Close daily cooperation between business people and developers | Frequent communication improves requirement understanding |
| 5 | Build projects around motivated individuals and trust them | Skilled teams should manage their own work |
| 6 | Face to face conversation is the most effective communication method | Direct communication reduces misunderstandings |
| 7 | Working software is the primary measure of progress | Progress is measured by functional features |
| 8 | Sustainable development pace | Teams should maintain a constant work pace |
| 9 | Continuous attention to technical excellence and good design | High quality design improves maintainability |
| 10 | Simplicity is essential | Developers should avoid unnecessary work |
| 11 | Best designs emerge from self organizing teams | Teams collaborate to find optimal solutions |
| 12 | Teams regularly reflect and improve their processes | Continuous improvement increases productivity |

These principles guide Agile development and ensure flexibility, collaboration, and quality.

---

## 6 Iteration in Agile Development

An iteration is a short development cycle where a small portion of the system is designed, developed, tested, and delivered.

Typical iteration length

1 to 4 weeks

Each iteration usually includes

• Planning  
• Design  
• Coding  
• Testing  
• Delivery

Example

| Iteration | Feature Delivered |
|---|---|
| Iteration 1 | Login system |
| Iteration 2 | User profile |
| Iteration 3 | Messaging system |

Each iteration produces a **working product increment**.

---

## 7 Incremental Development

Incremental development means building the system gradually by adding new features over time.

Instead of delivering the entire system at once, Agile delivers multiple small versions.

Example

Online shopping system development

Version 1  
User registration

Version 2  
Product search

Version 3  
Shopping cart

Version 4  
Payment system

This approach allows developers to test features earlier and improve them based on user feedback.

---

## 8 Agile Umbrella

The Agile umbrella refers to a group of development methodologies that follow Agile principles.

Examples of Agile methodologies

| Methodology | Description |
|---|---|
| Scrum | Most widely used Agile framework |
| Extreme Programming (XP) | Focus on coding quality and technical practices |
| Kanban | Visual workflow management |
| Crystal | Flexible team based development |
| Dynamic Systems Development Method (DSDM) | Structured Agile framework |
| Lean Software Development | Focus on reducing waste and improving efficiency |

Each methodology applies Agile principles in a different way.

---

## 9 User Stories

User stories are short descriptions of system requirements written from the user's perspective.

Standard format

As a [user role]  
I want [system function]  
So that [benefit]

Example

As a student  
I want to upload assignments  
So that my lecturer can evaluate my work

User stories help developers understand the user's needs and priorities.

---

## 10 Agile Development Process

Agile development follows a repeating cycle.

Typical Agile workflow

1 Requirement gathering  
2 Iteration planning  
3 Development  
4 Testing  
5 Customer feedback  
6 Improvement

This cycle repeats until the system is completed.

Example

| Iteration | Task |
|---|---|
| Week 1 | Login feature |
| Week 2 | Profile management |
| Week 3 | Messaging system |

---

## 11 Advantages of Agile Methodologies

| Advantage | Explanation |
|---|---|
| Faster delivery | Working software delivered early |
| Customer satisfaction | Continuous feedback improves product quality |
| Flexibility | Changes can be incorporated easily |
| Early error detection | Frequent testing identifies issues earlier |
| Better collaboration | Developers and stakeholders communicate regularly |

---

## 12 Disadvantages of Agile Methodologies

| Disadvantage | Explanation |
|---|---|
| Difficult cost estimation | Requirements change frequently |
| Requires experienced teams | Developers must manage their own work |
| Limited documentation | Some organizations require extensive documentation |
| Customer involvement required | Regular feedback is necessary for success |

---

## 13 Agile vs Structured Methodologies

| Feature | Structured Methodologies | Agile Methodologies |
|---|---|---|
| Development approach | Sequential | Iterative |
| Requirements | Fixed early | Changeable |
| Documentation | Heavy | Lightweight |
| Customer involvement | Limited | Continuous |
| Flexibility | Low | High |

Structured methodologies focus on strict planning while Agile focuses on adaptability.

---

## 14 Continuous Improvement

Agile teams continuously evaluate their development process.

This activity is called a retrospective.

During retrospectives teams discuss

• What worked well  
• What problems occurred  
• How to improve the next iteration

Continuous improvement ensures better performance in future iterations.

---

## Chapter Summary

Agile methodologies are modern development approaches that emphasize flexibility, collaboration, and rapid delivery of working software.

Key Agile concepts include

• Iterative development  
• Incremental delivery  
• Customer collaboration  
• Continuous improvement

The Agile Manifesto defines four core values and twelve principles that guide Agile practices.

Popular Agile methodologies include Scrum, Extreme Programming, and Kanban.
