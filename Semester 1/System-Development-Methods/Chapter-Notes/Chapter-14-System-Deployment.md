# Chapter 14 - System Deployment

---

## 1 Introduction to System Deployment

System deployment is the stage where the completed and tested system is delivered to the organization and begins actual operation. At this stage the system moves from the development environment into the real working environment where users will interact with it daily.

Deployment includes installing the system, configuring the operational environment, migrating data, training users, and selecting the strategy used to replace the old system.

Even if development and testing were successful, poor deployment planning may still cause system failure. Therefore deployment planning is an important part of system development.

---

## 2 Objectives of System Deployment

The deployment stage aims to ensure that the system becomes operational successfully.

| Objective | Explanation |
|---|---|
| Install the system | Transfer the developed system into the operational environment |
| Replace the old system | Transition from the previous system to the new one |
| Train users | Prepare users to operate the new system |
| Ensure smooth transition | Reduce disruption to business operations |
| Monitor system performance | Confirm the system works correctly in real conditions |

Deployment ensures that the system becomes usable by the organization.

---

## 3 Activities During System Deployment

Several activities occur during system deployment.

| Activity | Explanation |
|---|---|
| System installation | Install software and configure system components |
| Data migration | Transfer data from the old system to the new system |
| User training | Teach users how to operate the new system |
| System documentation delivery | Provide manuals and operational guides |
| System monitoring | Observe system performance after deployment |

These activities ensure the system operates properly in the real environment.

---

## 4 Data Migration

Data migration refers to transferring data from the old system to the new system.

This process is critical because incorrect data migration may cause system failure or incorrect information.

Steps in data migration usually include:

Extract data from the old system  
Clean and validate the data  
Convert data formats if necessary  
Load the data into the new system

Data accuracy must be verified after migration.

---

## 5 User Training

User training prepares employees to use the new system effectively.

Training ensures that users understand system functions and procedures.

Training methods include:

Workshops  
Hands on training sessions  
Training manuals  
Online tutorials

Well trained users improve system adoption and reduce operational errors.

---

## 6 System Documentation

System documentation is provided during deployment to help users and administrators understand the system.

Types of documentation include:

| Documentation Type | Description |
|---|---|
| User documentation | Instructions for system users |
| Technical documentation | Information for developers and technicians |
| Operational documentation | Instructions for system maintenance and operation |

Documentation helps ensure long term system maintenance.

---

## 7 System Changeover

System changeover refers to the process of replacing the old system with the new system.

Organizations must choose a strategy that minimizes risk and disruption.

Several changeover strategies exist.

---

## 8 Direct Changeover

Direct changeover replaces the old system with the new system immediately.

The old system is stopped and the new system starts operating at once.

Advantages:

Simple implementation  
Lower cost

Disadvantages:

High risk  
If the new system fails, operations may stop completely

Direct changeover is usually used for small systems or systems that are not critical.

---

## 9 Parallel Operation

Parallel operation means running the old system and the new system at the same time for a period.

Both systems process the same data and produce results simultaneously.

Advantages:

Low risk  
Results from both systems can be compared

Disadvantages:

Higher operational cost  
Extra effort required to maintain two systems

Parallel operation is often used for large or critical systems.

---

## 10 Pilot Operation

Pilot operation introduces the new system in a small part of the organization first.

Only one department or location uses the new system while others continue using the old system.

Advantages:

Reduced risk  
Problems can be detected early

Disadvantages:

Deployment takes longer  
Some departments may still rely on the old system

Pilot operation is useful for testing system performance in real conditions.

---

## 11 Phased Changeover

Phased changeover introduces the new system gradually.

Instead of replacing the entire system at once, individual modules or functions are implemented step by step.

Advantages:

Lower risk compared to direct changeover  
Gradual learning for users

Disadvantages:

Implementation takes longer  
Temporary integration between old and new components may be required

Phased changeover is suitable for complex systems with many modules.

---

## 12 Mirror Swapping for Maintenance

Mirror swapping is a technique used to perform system maintenance or updates without interrupting system operation.

A duplicate system called a mirror system is prepared.

Steps involved:

Create a complete duplicate of the main system  
Perform updates or maintenance on the mirror system  
After updates are completed, swap the mirror system with the main system

Advantages:

No downtime for users  
System maintenance becomes safer

Mirror swapping is commonly used for high availability systems.

---

## 13 Post Implementation Review

After deployment, organizations often perform a post implementation review.

The purpose is to evaluate whether the system meets the expected objectives.

The review examines:

System performance  
User satisfaction  
Operational efficiency  
Achievement of project goals

Feedback from this review may be used to improve future systems.

---

## 14 System Maintenance

After deployment the system enters the maintenance phase.

Maintenance involves updating and improving the system after it has been deployed.

Types of maintenance include:

| Maintenance Type | Description |
|---|---|
| Corrective maintenance | Fix system errors |
| Adaptive maintenance | Adjust system to new environments |
| Perfective maintenance | Improve system performance |
| Preventive maintenance | Prevent future problems |

Maintenance ensures that the system remains useful over time.

---

## Chapter Summary

System deployment is the stage where the completed system is delivered to the operational environment and begins actual use within the organization. During deployment activities such as system installation, data migration, user training, and documentation preparation are performed. Organizations must also choose a suitable changeover strategy to replace the old system with the new system. Common strategies include direct changeover, parallel operation, pilot operation, and phased changeover. Techniques such as mirror swapping can be used to perform system updates without interrupting system services. After deployment a post implementation review evaluates system performance and user satisfaction. The system then enters the maintenance phase where updates and improvements are made to ensure long term system effectiveness.
