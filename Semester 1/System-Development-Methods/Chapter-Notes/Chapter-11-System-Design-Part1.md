# Chapter 11 — System Design Part 1

---

## 1 Introduction to System Design

System design is the stage of system development where the requirements identified during system analysis are transformed into a detailed blueprint for building the system.

During system design, developers determine how the system will operate, how data will be processed, how system components will interact, and how users will interact with the system.

The goal of system design is to translate the logical requirements of the system into a structured plan that guides system implementation.

A well designed system ensures that the final system is efficient, reliable, secure, and easy to maintain.

---

## 2 Objectives of System Design

System design aims to convert system requirements into a detailed plan for system construction.

| Objective | Explanation |
|---|---|
| Transform requirements into design | Convert analysis results into system structure |
| Define system components | Identify modules, databases, and interfaces |
| Improve system performance | Ensure efficient system operation |
| Support system implementation | Provide a blueprint for developers |
| Ensure system quality | Design systems that are reliable and secure |

Effective system design helps reduce errors during development.

---

## 3 Logical Design and Physical Design

System design is commonly divided into two levels.

| Design Type | Description |
|---|---|
| Logical design | Describes how the system will function |
| Physical design | Describes how the system will be implemented |

Logical design focuses on system processes, data flow, and user interactions.

Physical design focuses on the technical details such as hardware, software, database structures, and network configuration.

Logical design is created first and serves as the basis for physical design.

---

## 4 System Architecture

System architecture defines the overall structure of the system and how system components interact with each other.

System architecture determines how different parts of the system work together to deliver system functionality.

A well designed architecture improves system performance, scalability, and maintainability.

Typical architecture components include:

• User interface layer  
• Application logic layer  
• Data management layer

These layers help separate different system responsibilities.

---

## 5 Types of System Architecture

Different system architectures can be used depending on system requirements.

| Architecture Type | Description |
|---|---|
| Standalone architecture | System runs on a single computer |
| Client server architecture | Processing tasks shared between client and server |
| Three tier architecture | System divided into presentation, application, and data layers |
| Web based architecture | System accessed through web browsers |
| Cloud based architecture | System hosted on cloud platforms |

Selecting the correct architecture ensures system flexibility and scalability.

---

## 6 Modular Design

Modular design divides a system into smaller components called modules.

Each module performs a specific task within the system.

Advantages of modular design include:

| Advantage | Explanation |
|---|---|
| Easier system development | Developers focus on smaller components |
| Easier maintenance | Modules can be modified independently |
| Improved system reliability | Errors are easier to isolate |
| Better testing | Individual modules can be tested separately |

Modular systems are easier to manage and maintain.

---

## 7 Structured Design

Structured design organizes system modules into a hierarchical structure.

This approach breaks complex systems into smaller and simpler components.

Structured design tools include:

| Tool | Purpose |
|---|---|
| Structure charts | Show relationships between modules |
| Hierarchy diagrams | Show module levels and dependencies |

Structured design helps developers understand system relationships clearly.

---

## 8 User Interface Design

User interface design focuses on how users interact with the system.

A good user interface improves usability and reduces user errors.

Key principles of interface design include:

| Principle | Explanation |
|---|---|
| Simplicity | Interface should be easy to understand |
| Consistency | Similar actions should follow similar design |
| Feedback | System should respond to user actions |
| Error prevention | Interface should reduce user mistakes |

User friendly interfaces improve user satisfaction and productivity.

---

## 9 Input Design

Input design focuses on how data is entered into the system.

Good input design ensures that data entered is accurate and complete.

Important input design considerations include:

• Input validation  
• Clear data entry forms  
• Proper error messages  
• Data format verification

Effective input design helps prevent incorrect data from entering the system.

---

## 10 Output Design

Output design focuses on how information is presented to users.

Outputs may include reports, dashboards, or printed documents.

Good output design ensures that information is easy to read and useful for decision making.

Key considerations include:

• Clear layout and formatting  
• Accurate information  
• Timely delivery of reports

Well designed outputs improve organizational decision making.

---

## 11 Program Design

Program design focuses on defining how software programs will perform system tasks.

Developers identify program modules, algorithms, and logic structures.

Program design tools include:

| Tool | Description |
|---|---|
| Pseudocode | Informal description of program logic |
| Flowcharts | Graphical representation of program steps |
| Structure charts | Show relationships between program modules |

Program design helps programmers implement system functionality correctly.

---

## 12 Design Specifications

Design specifications document the details of the system design.

Design documents usually include:

System architecture diagrams  
Database design  
Program specifications  
Interface design specifications  
Security design

These documents guide developers during system implementation.

---

## 13 Design Reviews

Design reviews are conducted to evaluate the quality of the system design before implementation begins.

During design reviews, stakeholders examine design documents to identify potential problems.

Benefits of design reviews include:

• Early identification of design errors  
• Improved system quality  
• Better communication among development team members

Design reviews help ensure that the system design meets project requirements.

---

## Chapter Summary

System design is the stage where system requirements are transformed into a detailed blueprint for system development. Designers determine the architecture, modules, interfaces, and programs required to build the system. System design includes logical design and physical design, which describe how the system will operate and how it will be implemented. Designers create system architectures, organize modules using structured design, and develop user interfaces that improve system usability. Input and output design ensure that data is entered accurately and information is presented clearly. Program design and design specifications guide developers during implementation, while design reviews ensure that the system design meets the required standards. A well planned system design provides the foundation for successful system implementation.
